<목표>
    제공된 코드의 외부 동작(입출력, 사이드 이펙트, UI 결과)은 완전히 동일하게 유지하면서,
    아래 측면을 개선하도록 리팩토링하세요.

    1. 가독성 — 코드를 처음 보는 팀원도 3분 안에 파악 가능한 수준
    2. 유지보수성 — 요구사항 변경 시 최소한의 코드만 수정하면 되는 구조
    3. 성능 — 불필요한 리렌더링, 메모리 누수, 무거운 연산 제거
    4. 타입 안전성 — any, unknown 남용 제거, 타입 추론 극대화
</목표>


<개선_사항>
    ## 1. 네이밍

    - 변수/함수명은 의도를 드러내는 동사+명사 조합으로 (예: fetchUserProfile, isLoading)
    - boolean 변수는 is/has/can/should 접두사
    - 이벤트 핸들러는 handle 접두사 (handlePress, handleSubmit)
    - 컴포넌트명은 PascalCase, 훅은 use 접두사 필수

    ## 2. 함수 분리 (단일 책임 원칙)

    - 함수 길이가 30줄 초과 시 반드시 분리
    - 한 함수는 하나의 추상화 레벨만 다룰 것
    - 조건 분기가 3depth 이상이면 early return 패턴 적용

    ## 3. 중복 제거

    - 동일 로직 2회 이상 반복 시 공통 유틸/훅으로 추출
    - 매직 넘버/문자열은 constants로 분리

    ## 4. React Native 성능

    - 불필요한 useMemo/useCallback 제거 (오버 최적화 금지)
    - 실제로 무거운 연산/객체 생성에만 메모이제이션 적용
    - FlatList keyExtractor 최적화, getItemLayout 가능 시 추가
    - useEffect 의존성 배열 정확히 명시, 클린업 함수 누락 확인

    ## 5. 메모리 누수 방지

    - useEffect 내 이벤트 리스너, 타이머, 구독은 반드시 cleanup 반환
    - AbortController로 비동기 요청 취소 처리
    - 컴포넌트 언마운트 후 setState 호출 방지

    ## 6. TypeScript 강화

    - any 타입 제거, 구체적인 타입 또는 제네릭으로 교체
    - interface보다 type alias 우선 (확장이 필요한 경우만 interface)
    - 옵셔널 체이닝(?.)과 nullish coalescing(??) 적극 활용
    - as 타입 단언 최소화, 타입 가드 함수로 대체
</개선_사항>

<폴더_아키텍처>
    - components/ 재사용 가능한 UI 컴포넌트 (props로만 동작, 외부 상태 의존 금지)
    - constants/ 매직 넘버, 정적 설정값, 라우트 경로 등 재사용 상수
    - contexts/ React Context (전역 상태, Provider 패턴)
    - hooks/ 커스텀 훅 (use 접두사, 로직 캡슐화)
    - pages/ 라우트 단위 화면 컴포넌트 (페이지 조합만 담당)
    - services/ API 호출, 외부 서비스 연동 (순수 함수 or 클래스)
    - types/ 공유 TypeScript 타입/인터페이스 정의
    - utils/ 순수 유틸리티 함수 (사이드 이펙트 없음, 테스트 용이)
    - validations/ 폼/데이터 검증 로직 (zod 스키마 등)
</폴더_아키텍처>


<주석_가이드>
    모든 함수, 커스텀 훅, 컴포넌트, 상수 위에 /\*\* \*/ 형식의 한 줄 JSDoc 주석을 작성하세요.

    // 올바른 예시
    - 사용자 프로필 정보를 서버에서 가져와 캐싱합니다
    const fetchUserProfile = async (userId: string) => { ... }

    - 입력값이 한국 휴대폰 번호 형식인지 검증합니다
    const isValidPhoneNumber = (value: string): boolean => { ... }

    - 결제 완료 후 주문 내역 페이지로 이동하는 버튼
    const PaymentSuccessButton = ({ onPress }: Props) => { ... }

    규칙:
    - 한 줄로 요약 (2줄 이상 금지)
    - "~합니다" 체로 끝낼 것
    - 자명한 코드에는 주석 생략 (i++, return null 등)
    - @param/@returns 태그는 복잡한 함수에만 선택적 추가
</주석_가이드>
